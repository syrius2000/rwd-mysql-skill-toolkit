## ADDED Requirements

### Requirement: 件数比較バリデーション

システム SHALL、フラットファイルの元件数・重複削除後の件数・DB 投入後の件数を比較し、整合性を検証する。

#### Scenario: 整合が取れている場合

- **WHEN** 元件数 − 重複件数 = DB 投入件数 である
- **THEN** バリデーションは成功し、サマリを出力する

#### Scenario: 整合が取れていない場合

- **WHEN** 上記の等式が成立しない
- **THEN** バリデーションは失敗し、各件数と差分をレポートする

#### Scenario: バリデーションレポートの出力

- **WHEN** バリデーションを実行する
- **THEN** 元件数、重複件数、投入件数、成功/失敗を一覧するレポートを出力する
