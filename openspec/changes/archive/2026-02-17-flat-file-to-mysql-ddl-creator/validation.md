# バリデーション手順（CP932 CSV → MySQL 投入）

当 change で定義するバリデーションのフローと成功/失敗の判定。3 ステップ（サンプル SQL＋レポート → 完成版 SQL（DB 名指定）→ 対象 DB へ実行）に沿う。

## フロー概要

0. **エンコーディング検証** … CP932 想定で検証する。失敗時はここで停止する。
1. **レコード数カウント** … 各 CSV ファイルのレコード数をカウントする（複数 CSV 対応）
2. **重複検出・レポート** … 重複がある場合を検出し、レポートする
3. **重複削除** … 重複しているレコードを削除し、ユニークなレコードのみの CSV を作成する
4. **MySQL 投入** … 完成版 SQL を指定した対象 DB へ実行し、ユニークなレコードを MySQL に投入する
5. **投入後件数カウント** … 投入後のテーブルのレコード数をカウントする
6. **件数比較** … 投入したレコード数と DB のレコード数を比較する
7. **報告**
   - 一致する場合: 成功を報告する
   - 一致しない場合: エラーを報告する

## 成功条件

- 元件数 − 重複件数 = 投入件数（DB のレコード数）が成立すること

## 失敗時

- 上記が成立しない場合はエラーとして報告し、元件数・重複件数・投入件数・差分をレポートに出力する。
