# Query Design Checklist

## 問いの分解

- 誰を対象にするか
- 何をイベント、曝露、アウトカム、属性として扱うか
- いつからいつまでを見るか
- 何をもって該当と判定するか
- 除外条件は何か

## 粒度

- 患者単位
- 受診単位
- 処方単位
- 検査単位
- イベント単位

粒度が異なるテーブルを JOIN すると行数が増える。JOIN 前後で `COUNT(*)` と `COUNT(DISTINCT PATIENTNO)` を確認する。

## 検証

- 総行数
- ユニーク ID 数
- NULL
- 重複
- 日付範囲
- カテゴリ値
- JOIN 前後の増減
- 解析対象外コードの混入

## 分析への引き渡し

- 一行の単位を書く
- 抽出条件を書く
- 未確認リスクを書く
- 推奨する次スキルを書く
