# .agent/skills 監査レポート (2026-04-04)

現在の `.agent/skills` 配下のスキル（当時 10 件、現行 **13** 件）を調査した結果、共通する課題と個別のバグ・訂正事項について報告します。

---

## 1. 全体的な課題

### ❌ 自動テストの完全な欠如（最優先事項）
すべてのスキルにおいて、**自動テスト（Unit Test / Integration Test）が一切実装されていません。**
- `tests/` ディレクトリが存在しない。
- `test.sh` や `Makefile` 等の検証用エントリーポイントがない。
- 現状、AI またはユーザーが「手動で実行して結果を目視する」ことでしか動作確認ができておらず、デプロイや改修時のデグレード検知が不可能です。

### ⚠️ 依存関係管理の曖昧さ
- 各スキルが特定の Python パッケージ（`pandas`, `jsonlite` 等）や R パッケージ（`vcd`, `gt` 等）に依存していますが、スキル単位での `requirements.txt` や `renv.lock` がありません。
- 実行環境の差異により、パスが通っていてもライブラリ不足で実行に失敗するリスクがあります。

---

## 2. 個別スキルのバグ・修正事項

### 🛠 mysql-table-cardinality
- **脆弱性 (Injection Risk)**: `get_cardinality_cli.py` において、`database` や `table` 名をバックティックで括る（L181-183）だけの簡易的なエスケープが行われています。入力値に対する厳格なバリデーションがなく、悪意のあるメタ文字が含まれた場合に誤動作する可能性があります。
- **エラーハンドリング**: `_run_mysql` で `check=False` としているため、DB 接続エラー時にスタックトレースがユーザーに分かりにくく、ログレベルの制御も不十分です。

### 🛠 vcd-categorical-analysis
- **堅牢性**: 最近の改修で CLI 引数対応（`--data` 等）を強化しましたが、入力データの「型」チェックが不足しています。
  - 数値型として読み込まれたカテゴリ変数を自動的に factor に変換する機能がないため、解析が意図せず連続値として扱われ、`loglm` がエラーを吐く可能性があります。
- **スパースデータへの対応**: 疎密度（Sparsity）が高い場合、`loglm` が収束しないケースがありますが、その際の「警告」や「代替手段（Fisherの正確検定への切り替え）」のロジックが未実装です。

### 🛠 questionnaire-batch-analysis
- **実行継続性**: `batch_runner.R` は `tryCatch` でエラーを捕捉していますが、特定の設問でエラーが起きた際、`summary.csv` に失敗理由は残るものの、**「なぜ失敗したか（どのパラメータが悪かったか）」**のデバッグ情報が不足しています。
- **設定依存**: `question_config.csv` の文字エンコーディングが CP932 の場合に読み込みに失敗する可能性があり、`fileEncoding` の指定が漏れています。

## 3. 推奨アクションプラン

### 短期（即時修正可能）
1. **各スキルへの `tests/` 設置**:
    - 最小限のテスト用 CSV と、それを実行して成果物の存在を確認する `verify_skill.sh` を作成。
2. **`vcd-categorical-analysis` のデータクレンジング強化**:
    - 全列を `as.factor` する、あるいは数値型列が含まれる場合に警告を出す処理の追加。

### 中長期
1. **共通ユーティリティの導入**:
    - SQL インジェクション対策やロギング、パス解決（`_find_repo_root`）など、複数のスキルで重複しているコードを統合し、共通モジュール化する。
2. **Dockerfile / DevContainer の提供**:
    - R や Python の依存環境を固定し、「誰の環境でも動く」ことを保証する。

---

> [!IMPORTANT]
> 「テスト未実装」については、特に **`vcd-categorical-analysis`** のような複雑な統計計算を伴うスキルで致命的です。基準となるデータセット（HairEyeColor 等）を用いた期待値との比較テストを優先的に実装することを推奨します。
