# AnotherPJ — RDBMSインポートおよびデータ重複排除ツール

created: 2026-06-03 22:42 (JST)
author: AI Agent (Gemini 3.5 Flash)

## 旧式プロンプト事例: CSVインポート＆重複排除SQL生成 (統合版)

かつてVS CodeのGitHub CopilotプロンプトやAIエージェントの指示用として使用していた、CSVのインポートと重複排除を一度に行うSQLを生成するための統合プロンプトの事例です。

```markdown
---
title: "CSVインポートおよび重複排除SQL生成 (統合版)"
description: "CSVサンプルを基に、一時テーブルへのロード、重複排除、最終テーブルの作成、重複確認用SQLを生成します。"
mode: agent
---

### 目標・目的
あなたはプロのデータベースエンジニアです。
CSVファイルをデータベースに取り込み、重複を取り除いたクリーンなテーブルを作成するための一連のSQL文を生成してください。

### ワークフロー
1. **一時テーブルの作成**: CSV構造に基づいた `CREATE TABLE` 文を生成（テーブル名: `[ファイル名]_staging`）。
2. **データロード**: 一時テーブルへCSVを投入する `LOAD DATA INFILE` 文を生成。
3. **重複排除と本番テーブル作成**: `SELECT DISTINCT` を用いて、重複を除いた本番テーブルを作成（テーブル名: `[ファイル名]`）。
4. **重複行・差分の比較検証 (参考)**: どのデータが重複していたかを検出するクエリ、および新旧テーブルを比較するクエリを生成（※コメントアウト）。
5. **クリーンアップ**: 一時テーブルを削除する `DROP TABLE` を生成（※検証のためコメントアウト）。

### 要件
1. **データベース名**: `VACCINE`
2. **対象テーブル名**: `{{file}}` のファイル名（拡張子なし）をベースにする（例: `myTableName`）。
    - 一時テーブル: `[テーブル名]_staging`
    - 本番テーブル: `[テーブル名]`
3. **データ型推測**:
    - 日付型 (`DATE`/`DATETIME`) のみ推測して型定義し、それ以外は `VARCHAR(255)` とする。主キーやインデックスは不要。
4. **日付データのSTR_TO_DATE処理**:
    - `LOAD DATA INFILE` 内で `STR_TO_DATE(NULLIF(@var, ''), '【日付書式】')` を使用し安全に日付変換を行う。
5. **重複検証クエリの作成**:
    - 重複していた行を特定するクエリ `SELECT *, COUNT(*) FROM [一時テーブル] GROUP BY [全カラム] HAVING COUNT(*) > 1;` を参考用に配置する。
6. **テーブルクリア処理**:
    - 再実行時のため、作成前には `DROP TABLE IF EXISTS` を配置する。
```
