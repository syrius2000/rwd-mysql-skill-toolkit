# Requirements: vcd-ai-evaluation-report

## 1. 概要 (Overview)
現在の `vcd-categorical-analysis` スキルは R のスクリプト（または RMarkdown）により、データ読み込みから結果図表の出力までを一気通貫で処理している。これを改良し、スクリプトは「AIが解釈しやすい小さな結果データファイル（JSON、CSV等）および画像ファイル」を出力する処理に留め、最終的なレポート作成と結果の評価・解釈は AI が担う設計に変更する（AI駆動による報告へと移行する）。

## 2. 目的と価値 (Business Value)
* AIによるコンテキストに応じた考察（$p$値やCramer's Vの解釈、最大の乖離を見せるセルの考察手法等）を第一章に据えることで、単なる表とグラフから「インサイトや結論」を直接得られるレポート体験を提供する。
* レポートフォーマットが統一化され（第一章：評価、第二章：残差サマリ、第三章：図表）、利用者が理解しやすい順序で情報にアクセスできる。

## 3. 機能要件 (Functional Requirements)
1. **個別結果ファイル出力機能の実装**
   * R スクリプト（例：`templates/analysis.R`）を改修し、Cramer's V、$P$値、ディビアンス、最大残差のセルなどの主要指標を含むサマリー用JSONファイルを出力する。
   * AIによる詳細な検証用途に対応するため、セル別の詳細な残差データをCSVファイルとして出力する。
2. **AIによる章立てレポート要件**
   * **第一章**: AIが出力データ（JSON/CSV）を読み取り、統計的妥当性や業務的インサイトを含む「AI評価コメント」を作成する。
   * **第二章**: AIが妥当と判断した結果モデル（主効果モデル等）の残差表やサマリーデータを提示する。
    このとき、モデル診断結果から妥当なモデルが決定されるが、複雑なモデルは解釈困難なため、glmの残渣分析プロットや表は加法モデル（主効果モデル）のものを提示する。
   * **第三章**: 解析のために作成された画像（Mosaicプロットや残差プロット等）を参考図表として組み込む。
3. **スキルドキュメントの更新**
   * エージェントがRスクリプトから結果を集め、自律的にレポートを生成する手順（ワークフロー）が `SKILL.md` および `references/workflow.md` に明記されていること。

## 4. 非機能要件 (Non-functional Requirements)
* 出力するJSONやCSVは、AI（LLM）がトークン効率よくかつ正確にパースしやすいシンプルなテーブル構造（ネストを抑えたキーバリューなど）を保つこと。
* Rスクリプト自体が重いHTMLレンダリングなどを伴わず、迅速に終了すること。

## 5. 受け入れ基準 (Acceptance Criteria)
- [ ] Rスクリプトが JSON 、 CSV、画像ファイルを `./skill_out/vcd_categorical/` にエラーなく正しく出力している。
- [ ] 出力されたこれらの中間ファイルを用いて、AIが「第一章（AI評価）、第二章（残差・サマリ）、第三章（参考図表）」という構成を持つ Markdown レポート（Artifactなど）を矛盾なく出力できる。
- [ ] スキルガイド類 (`SKILL.md`, `workflow.md`) が、これら「AI主導の章立て出力フロー」を前提とした記載に更新されている。
