# Implementation Tasks: vcd-ai-evaluation-report

## 1. Rモデリングロジックの改修
- [ ] 1.1. **パッケージ参照と名前空間の徹底**: `pacman::p_load` に `jsonlite` を追加し、コード内の全関数呼び出しを `stats::chisq.test` などのように `namespace::function` 形式へリファクタリングする。
- [ ] 1.2. **2段階ポアソンモデルの実装**: 3-way解析時に「主効果モデル (`A+B+C`)」と「2-way交互作用モデル (`(A+B+C)^2`)」の両方を適合（fit）させるロジックを組む。
- [ ] 1.3. **スパースデータのエラーハンドリング**: `stats::anova()` 等がスパース要因（0が多い等）でクラッシュ・エラーを発火した際の `tryCatch` を用いたセーフティフォールバック（不適合時は該当モデル関連数値を `NA` や `0` で安全にスキップする）を実装する。

## 2. フットプリント（JSON/CSV）出力機能の追加
- [ ] 2.1. **サマリー JSON の出力 (`summary.json`)**: 各モデルの逸脱度 (Deviance)、自由度、P値、Cramer's V、および各種最大残差を持つセル名と値を抽出し、`jsonlite::write_json` で出力する。
- [ ] 2.2. **残差 CSV の最適化と出力**: `pearson_res` に加えて `abs_pearson_res` 列を追加。全件生データは `residuals_raw.csv` に出力しつつ、AIのトークン効率向上のため上位20行程度や特出するものに絞り `abs_pearson_res` 降順にソートした `residuals_significant.csv` を主力として出力させる。

## 3. スキルドキュメント・AIプロンプトの更新
- [ ] 3.1. **`references/workflow.md` の更新**: AI（LLM）が自律で `analysis.R` を実行し、生成された小結果ファイルを回収してレポートを組み立てる「2段階評価アーキテクチャ」の Mermaid シーケンスを記述する。
- [ ] 3.2. **`SKILL.md` のプロンプト指示調整**: AIの評価フェーズにおいて、「第1段階: 主効果からの全体相関の俯瞰（自明な関連性）」「第2段階: 2-way交互作用残差からの局所・特異な依存関係（Simpsonパラドックス等）」の洞察を徹底的に指示するプロンプトを明記する。さらに、これらを「第1章〜第3章」のArtifactフォーマットで美しく出力させるよう縛りを加える。
