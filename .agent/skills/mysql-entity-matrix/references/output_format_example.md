# Output Format Reference for mysql-entity-matrix

This reference defines the output format mapping generated by `scripts/generate_matrix_sql.py`.

## Expected Output Matrix (Pattern A)

When identifying all unique IDs out of a given column (`PATIENTNO` by default) across an entire database, the skill logic generates SQL ending with a standard format matching Pattern A. The first column is the designated identifier. All subsequent columns represent one evaluated table, holding `1` or `0` flag values.

| PATIENTNO   | TABLE_A | TABLE_B | TABLE_C | TABLE_D | ...     |
| :---        | :---    | :---    | :---    | :---    | :---    |
| P0001       | 1       | 1       | 0       | 0       | ...     |
| P0002       | 1       | 0       | 1       | 1       | ...     |
| P1045       | 0       | 1       | 1       | 0       | ...     |

### Underlying Generated SQL Pattern

The generated SQL involves standard Common Table Expressions (CTE) dynamically generated to UNION all target tables finding unique IDs, and then running a comprehensive `LEFT JOIN` and `CASE` aggregation block.

```sql
WITH all_distinct_ids AS (
    SELECT id_column AS ID FROM table_a
    UNION
    SELECT id_column AS ID FROM table_b
    UNION
    SELECT id_column AS ID FROM table_c
    -- ... dynamically added unions
)
SELECT
    base.ID AS TARGET_ID,
    CASE WHEN t_a.id_column IS NOT NULL THEN 1 ELSE 0 END AS table_a,
    CASE WHEN t_b.id_column IS NOT NULL THEN 1 ELSE 0 END AS table_b,
    CASE WHEN t_c.id_column IS NOT NULL THEN 1 ELSE 0 END AS table_c
    -- ... dynamically added case blocks
FROM all_distinct_ids base
LEFT JOIN (SELECT DISTINCT id_column FROM table_a) t_a ON base.ID = t_a.id_column
LEFT JOIN (SELECT DISTINCT id_column FROM table_b) t_b ON base.ID = t_b.id_column
LEFT JOIN (SELECT DISTINCT id_column FROM table_c) t_c ON base.ID = t_c.id_column
-- ... dynamically added joins
ORDER BY base.ID;
```
