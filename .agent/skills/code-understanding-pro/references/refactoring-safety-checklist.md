# リファクタリング安全性チェックリスト

## 原則

リファクタリングは、原則として外部挙動を変えない改善である。
挙動変更を伴う場合は、リファクタリングではなく仕様変更または機能変更として明示する。

## 必須確認

- [ ] 挙動変更の有無を明示したか
- [ ] 入力/出力の契約は変わらないか
- [ ] 例外の種類や発生条件は変わらないか
- [ ] ログ、監査証跡、メトリクスは維持されるか
- [ ] DB更新、ファイルI/O、API呼び出し順序は変わらないか
- [ ] 並行処理や非同期処理の順序依存は壊れないか
- [ ] 時刻、乱数、環境変数依存の挙動は維持されるか
- [ ] 既存テストは通るか
- [ ] 追加すべき回帰テストを示したか
- [ ] ロールバック方針を示したか

## 変更前後比較テンプレート

````markdown
## リファクタリング案

### 目的
...

### 挙動変更
- なし / あり
- ありの場合、その内容:

### 変更前
```python
...
```

### 変更後
```python
...
```

### 利点
- ...

### リスク
- ...

### 必要なテスト
- ...

### 代替案
- ...
````

## 行数削減に関する注意

短いコードは必ずしも良いコードではない。
以下を削ってまで短くしてはいけない。

- バリデーション
- 型変換
- エラー処理
- ログ
- 監査証跡
- テスト容易性
- 意図を示す中間変数
- ドメイン上重要な分岐
