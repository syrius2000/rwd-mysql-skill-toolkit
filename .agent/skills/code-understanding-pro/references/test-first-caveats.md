# テスト優先確認の注意点

## 基本方針

テストは開発者の意図、仕様、境界条件を理解するための重要な証拠である。
ただし、テストは絶対ではない。

## テストから得られる情報

- 期待される正常系の振る舞い
- 異常系の扱い
- 境界値
- ユースケース
- 過去に発生したバグ
- API契約
- 入出力例

## テストを疑うべき状況

- テスト名と検証内容が一致しない
- 実装詳細を固定している
- snapshotが過剰に広い
- 異常系がない
- テストデータが現実的でない
- 古い仕様を前提としている
- 失敗テストが放置されている
- テストがmockだらけで実際の副作用を検証していない

## テスト評価テンプレート

```markdown
## テスト評価

### 見つかったテスト
| ファイル | 対象 | 検証している振る舞い |
|---|---|---|

### 良い点
- ...

### 不足
- ...

### 実装詳細依存の可能性
- ...

### 追加すべきテスト
| ケース | 目的 | 期待結果 |
|---|---|---|
```
