# レビュー重要度ガイド

## [Critical]

マージ不可。即時対応が必要。

例：

- セキュリティ脆弱性
- データ消失
- 個人情報、認証情報の漏洩
- 重大な計算誤り
- 契約/API仕様の破壊
- 本番障害につながる明白なバグ

出力例：

```markdown
- **[Critical] 認証なしで管理APIにアクセス可能**
  - 場所: `src/api/admin.py:L42-L55`
  - 根拠: 認証middlewareを通らずに削除処理が呼ばれている
  - なぜ重要か: 権限のないユーザーがデータ削除できる
  - 修正案: 認証/認可チェックを追加し、既存middlewareへ統合する
  - 実行すべきテスト: 権限なし、権限あり、期限切れtokenの3ケース
```

## [Major]

強く修正推奨。放置するとバグ、保守困難、テスト不能につながる。

例：

- 境界値処理不足
- 欠測/null処理不足
- 重要テスト不足
- 例外処理の欠落
- 性能劣化の可能性が高い実装
- 主要ユースケースで壊れる可能性

## [Consider]

検討価値あり。設計改善、可読性、拡張性の提案。

例：

- 責務分離
- 関数分割
- 依存注入
- 命名改善
- データ構造の整理
- 重複ロジックの統合

## [Nit]

任意修正。局所的な読みやすさやスタイルの問題。

例：

- 小さな命名改善
- import順
- フォーマット
- コメント表現

## [FYI]

情報提供のみ。アクション不要。

例：

- 代替ライブラリ情報
- 将来の改善候補
- 周辺仕様の注意
- 知識共有

## 指摘テンプレート

```markdown
- **[分類] タイトル**
  - 場所:
  - 根拠:
  - なぜ重要か:
  - 修正案:
  - 実行すべきテスト:
```
