## 4. パフォーマンス・品質評価

- **[CRITICAL]** 重大バグ / 行数爆発リスク
  - 例: 多対多JOINによる行数の重複
- **[PERFORMANCE]** コスト・パフォーマンス評価
  - 例: パーティションキーの非活用、フルスキャン
- **[EDGE_CASE]** NULL / 例外ハンドリング
  - 例: `COALESCE` 未適用による計算結果の NULL 化
