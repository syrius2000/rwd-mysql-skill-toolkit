# 分析品質契約

この文書は、VCD系およびquestionnaire系スキルで共通して使う品質契約です。Rエンジン本体の代替ではなく、Pass 0から完了報告までの判断、AIレビュー、成果物確認を揃えるために使用します。

## 適用対象

- `.agent/skills/vcd-pass0-consultation/`
- `.agent/skills/vcd-categorical-analysis/`
- `.agent/skills/vcd-bayesian-evidence-analysis/`
- `.agent/skills/questionnaire-batch-analysis/`

## Pass別ゲート

| Pass | 確認対象 | 成果物または確認 |
| :--- | :--- | :--- |
| Pass 0 | 入力、分析問い、変数、水準、欠損、度数列、過剰次元、セルスパース性 | `inspection_results.json`, `data_analysis_scope.md`, `analysis_config.json` |
| Pass 1 | 統計計算、設定、出力ディレクトリ、run分離、主要JSON/CSV | R実行ログ、`summary*.json`, `evidence_results.json`, `summary.csv` |
| Pass 2 | AIレビュー、統計指標の読み分け、結論、根拠、限界、解釈保留、次アクション | `executive_summary.md` または設問別レビュー |
| Pass 2.5 | 過剰主張、P値偏重、図表と本文の矛盾、スパースセル、大標本効果、情報損失 | `quality_check.md` |
| Pass 3 | HTML/ダッシュボード、図表、表、本文、完了報告 | `dashboard.html`, `report.html`, `cross_question_summary.md` |

## データ品質とスコープ

統計計算前に以下を確認します。

- 入力CSV、設定ファイル、設問設定CSVが存在する。
- 分析対象変数と度数列が入力データに存在する。
- 欠損、過剰水準、極端に少ないセル、4次元以上の投入、設問設定の不整合を確認する。
- 集約、除外、層別、再分類を行う場合は、情報損失と解釈への影響を明示する。
- 不整合が分析結果を変える場合は、実行せず修正または確認を求める。

## AIレビュー標準構成

AIレビュー成果物は日本語で記載し、最低限以下を含めます。

1. 主要結論: 何が分かったかを先に書く。
2. 根拠: 効果量、残差、Evidence Score、Bayes Factor、セル数、サンプルサイズを区別して示す。
3. 限界: 欠損、スパースセル、過剰水準、大標本効果、集約による情報損失を必要に応じて示す。
4. 解釈保留: 結論を断定できない設問、セル、層、設定不整合を明示する。
5. 次アクション: 再分類、層別、追加確認、報告上の注意点を示す。

## 禁止・保留表現

- P値のみで結論しない。
- Evidence Scoreが負のセルを「有意な関連あり」と表現しない。
- 残差方向を確認せずに「多い」「少ない」と書かない。
- 時間順序や介入情報がない結果から因果を断定しない。
- 大標本で効果量が小さい結果を、実務的に強い発見として扱わない。
- `status=error`、設定不整合、解析不能な設問を結果解釈に混ぜない。

## 可視化QA

図表は特定の主張を支える証拠として扱います。

- 図表のタイトル、ラベル、凡例、単位、符号、色分けが本文と整合している。
- 残差やEvidence Scoreの正負、上位セルの順位、設問タイプが本文と矛盾しない。
- 長いラベルや多数水準により読めない場合は、集約、表形式、保留事項のいずれかを選ぶ。
- 図表が主張を支えない場合は、本文を修正するか、図表の役割を保留事項として記録する。

## Pass 2.5品質確認

Pass 2の後、必要に応じて同じrun出力ディレクトリに`quality_check.md`を保存します。少なくとも以下を確認します。

- AIレビューが標準構成を満たす。
- P値偏重、因果断定、サンプルサイズやセル数の無視がない。
- 大標本効果、スパースセル、過剰水準、情報損失が必要に応じて明示されている。
- 図表、表、JSON/CSVの値と本文が矛盾していない。
- 重大な未解決事項がある場合は、完了報告でブロッカーまたは解釈保留として示す。

## questionnaire横断総括

`questionnaire-batch-analysis`で複数設問を扱う場合は、設問別成果物を置き換えず、上位索引として`cross_question_summary.md`を生成できます。

- `summary.csv`と設問別`report.html`を根拠にする。
- 重要設問、解釈保留、実務的示唆、次アクションを分ける。
- P値だけで設問を順位付けしない。
- 効果量、残差方向、セル数、設問タイプ、エラー状態を合わせて判断する。
- `nominal_2way`、`likert_2way`、`nominal_3way`の解釈差を明示する。

## 完了条件

完了報告では以下を示します。

- 主要成果物、補助成果物、確認用成果物の保存先。
- 主要成果物が存在し、読める状態であること。
- 重大な入力品質、統計解釈、図表整合、レンダリング問題が残っていないこと。
- 残る場合は、完了ではなくブロッカーまたは解釈保留として報告する。

## 非対象

この契約はMCP artifact、Reactウィジェット、外部Data Analyticsランタイムを要求しません。既存のローカルR/Markdown/HTML中心の実行を維持します。
